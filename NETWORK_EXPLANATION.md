# ОБЪЯСНЕНИЕ СЕТЕВОЙ АРХИТЕКТУРЫ

## ❓ Вопрос: Что происходит при нажатии 'T'?

### Краткий ответ:
**НЕТ**, нажатие 'T' **НЕ активирует** сетевые функции автоматически!

При нажатии 'T' запускается **ТОЛЬКО локальная симуляция**. Данные отправляются по сети **ТОЛЬКО ЕСЛИ** сервер уже запущен (Ctrl+S).

---

## Подробное объяснение

### 1. Нажатие клавиши 'T' (Создание симулированной машины)

```cpp
// В main.cpp:
if (key == GLFW_KEY_T && action == GLFW_PRESS) {
    static int vehicle_id = 1;
    g_vehicles[vehicle_id] = Vehicle(vehicle_id);
    simulateVehicleMovement(vehicle_id, g_smooth_track_points);  // ◄── Запускает поток
    vehicle_id++;
}
```

**Что происходит:**
1. Создается объект `Vehicle` с уникальным ID
2. Запускается фоновый поток `simulationThreadWorker()`
3. Этот поток **работает локально** на вашем ПК

---

### 2. Работа симуляции (фоновый поток)

```cpp
static void simulationThreadWorker(int vehicle_id, std::vector<SplinePoint> smooth_path)
{
    while (true)
    {
        // 1. Вычисляем физику (скорость, позицию)
        updateVehiclePhysics(...);
        
        // 2. Интерполируем позицию на треке
        glm::vec2 pos = interpolateTrackPosition(...);
        
        // 3. Конвертируем в GPS
        convertNormalizedToGPS(...);
        
        // 4. Создаем пакет данных
        TelemetryPacket packet = createTelemetryPacket(...);
        
        // 5. ✅ Обрабатываем ЛОКАЛЬНО
        processIncomingTelemetry(packet);  // ◄── ЗДЕСЬ МАГИЯ!
        
        sleep(50ms);
    }
}
```

**Важно:** `processIncomingTelemetry()` - это **единая точка обработки**, она:
- ✅ Обновляет `g_vehicles[id]` (локально, всегда)
- ✅ Вызывает `BroadcastTelemetryToClients()` (отправляет по сети, ТОЛЬКО ЕСЛИ сервер запущен)

---

### 3. Функция processIncomingTelemetry()

```cpp
void processIncomingTelemetry(const TelemetryPacket& packet)
{
    // ✅ 1. ЛОКАЛЬНАЯ ОБРАБОТКА (всегда выполняется)
    {
        std::lock_guard<std::mutex> lock(g_vehicles_mutex);
        
        // Обновляем или создаем машину в g_vehicles
        auto it = g_vehicles.find(packet.ID);
        if (it != g_vehicles.end()) {
            // Обновляем координаты, скорость и т.д.
            vehicle.m_lat_dd = packet.lat / 1e7;
            vehicle.m_lon_dd = packet.lon / 1e7;
            // ...
        } else {
            // Создаем новую машину
            g_vehicles[packet.ID] = Vehicle(packet);
        }
    }
    
    // ✅ 2. СЕТЕВАЯ РАССЫЛКА (выполняется ТОЛЬКО ЕСЛИ сервер запущен!)
    BroadcastTelemetryToClients(packet);  // ◄── ПРОВЕРЯЕТ isServerRunning()
}
```

---

### 4. Функция BroadcastTelemetryToClients()

```cpp
void BroadcastTelemetryToClients(const TelemetryPacket& packet)
{
#if NETWORKING_ENABLED
    // ❌ GUARD CLAUSE: Если сервер НЕ запущен → ничего не делаем!
    if (!g_is_server_running) return;  // ◄── КЛЮЧЕВАЯ ПРОВЕРКА!
    
    // Проверяем наличие подключенных клиентов
    bool has_clients = false;
    for (const auto& pair : g_authenticated_connections) {
        if (pair.second) {
            has_clients = true;
            break;
        }
    }
    
    // ❌ Если клиентов нет → ничего не отправляем
    if (!has_clients) return;
    
    // ✅ Отправляем пакет всем клиентам
    SendToAll(&packet, sizeof(TelemetryPacket));
#endif
}
```

**Важно:** Эта функция **НИЧЕГО НЕ ДЕЛАЕТ**, если:
- Сервер не запущен (`g_is_server_running == false`)
- Нет подключенных клиентов

## Схема работы (пошагово)

### Сценарий 1: Только локальная симуляция (БЕЗ сервера)

```
1. Нажимаем 'T'
   └─> Создается машина с ID=1
   └─> Запускается simulationThreadWorker(1, track)

2. Поток симуляции работает:
   └─> updateVehiclePhysics() - вычисляет физику
   └─> createTelemetryPacket() - создает пакет
   └─> processIncomingTelemetry(packet)
       └─> g_vehicles[1] обновляется ✅
       └─> BroadcastTelemetryToClients()
           └─> g_is_server_running == false
           └─> return; (ничего не отправляет) ❌

3. Результат:
   ✅ Машина движется на экране (локально)
   ❌ Данные НЕ отправляются по сети
```

### Сценарий 2: Локальная симуляция + Сервер

```
1. Запускаем сервер (Ctrl+S)
   └─> g_is_server_running = true
   └─> Открывается порт 777
   └─> Ждет подключения клиентов

2. Нажимаем 'T' (создаем машину)
   └─> simulationThreadWorker() запускается
   └─> processIncomingTelemetry(packet)
       └─> g_vehicles[1] обновляется ✅
       └─> BroadcastTelemetryToClients()
           └─> g_is_server_running == true ✅
           └─> Есть клиенты? Проверяем...
               └─> Если ДА → SendToAll() ✅
               └─> Если НЕТ → return ❌

3. Результат:
   ✅ Машина движется на экране (локально)
   ✅ Данные отправляются клиентам (если они подключены)
```

### Сценарий 3: Реальные данные с ESP32 + Сервер

```
1. Запускаем сервер (Ctrl+S)
   └─> g_is_server_running = true

2. Запускаем чтение COM-порта (клавиша 'R')
   └─> startRealDataCapture("COM3")
   └─> realDataThreadWorker() запускается
   └─> Читает данные из COM3

3. ESP32 отправляет пакет:
   └─> serial.readBytes() получает данные
   └─> processIncomingTelemetry(packet)
       └─> g_vehicles[packet.ID] обновляется ✅
       └─> BroadcastTelemetryToClients()
           └─> SendToAll() отправляет клиентам ✅

4. Результат:
   ✅ Реальная машина отображается на экране
   ✅ Данные с ESP32 рассылаются клиентам
```

## Интеграция с существующими сетевыми функциями

### ✅ Что ОСТАЛОСЬ БЕЗ ИЗМЕНЕНИЙ:

1. **Запуск сервера** (Ctrl+S):
   ```cpp
   if (key == GLFW_KEY_S && action == GLFW_PRESS && g_control == GLFW_PRESS) {
       if (!isServerRunning()) {
           std::thread server(serverWork);
           server.detach();
           ChangeisServerRunning();
       }
   }
   ```
   - Работает так же, как раньше
   - Открывает порт 777
   - Ждет подключения клиентов

2. **Клиент** (`Client.cpp`):
   - Подключается к серверу по IP:порт
   - Аутентифицируется паролем
   - Получает пакеты через `recvfrom()`

3. **Аутентификация клиентов**:
   - `g_authenticated_connections` - список подключенных клиентов
   - Работает так же, как раньше

### ✅ Что ИЗМЕНИЛОСЬ:

1. **Содержимое пакетов**:
   - Раньше: `RandomTelemetryData()` - случайные данные
   - Теперь: **реальные** данные от симуляции или ESP32

2. **Отправка через единую точку**:
   - Раньше: сервер генерировал и отправлял данные в цикле
   - Теперь: `processIncomingTelemetry()` автоматически отправляет при получении пакета

## Ответы на частые вопросы

### ❓ "При нажатии 'T' активируются сетевые функции?"
**Ответ:** НЕТ. 'T' только создает локальную машину. Отправка по сети происходит ТОЛЬКО ЕСЛИ сервер запущен.

### ❓ "Это работает с моими текущими сервером/клиентом?"
**Ответ:** ДА! Сервер и клиент работают ТАК ЖЕ, как раньше. Изменилось только СОДЕРЖИМОЕ пакетов (теперь реальные данные вместо случайных).

### ❓ "Можно ли работать без сервера?"
**Ответ:** ДА! Симуляция работает локально. Сервер нужен ТОЛЬКО для передачи данных другим ПК.

### ❓ "Можно ли одновременно симуляцию и ESP32?"
**Ответ:** ДА! Обе системы обрабатываются через `processIncomingTelemetry()`, поэтому работают одновременно.

### ❓ "Как понять, отправляются ли данные по сети?"
**Ответ:** Проверь консоль сервера:
```
[SERVER] Ready to broadcast telemetry from real/simulated vehicles
[SERVER] Telemetry will be sent via processIncomingTelemetry() -> BroadcastTelemetryToClients()
```
Если видишь это сообщение → сервер работает и рассылает данные.

# RAJAGP Client

**RAJAGP** — система хронометража и телеметрии для гонок радиоуправляемых машин и картинга в реальном времени. Этот репозиторий — десктопный клиент под Windows (x64 / ARM64): отрисовка карты трека и машин, тайминги кругов и секторов, управление гонкой и профессиональный экран Pit Wall.

Сайт: [rajagp.eu](https://rajagp.eu/) · Издатель: RAJAGP SIA · Текущая версия: **1.1**

---

## Экосистема

Клиент — одна из четырёх частей системы:

| Компонент | Описание |
|---|---|
| **RAJAGP Client** (этот репозиторий) | Приложение оператора: визуализация, тайминг, управление гонкой |
| **RAJAGP Server** (закрытый репозиторий) | Track Server: приём телеметрии от трекеров, авторитетный расчёт позиций/кругов, раздача данных клиентам по WebSocket. Содержит общую библиотеку протокола `rajagp_core` |
| **Трекер (передатчик)** | Бортовой модуль на машине (ESP32 + GPS/RTK): собирает координаты и телеметрию, передаёт их по радиоканалу LoRa (SX1280) |
| **Приёмник** | Наземный модуль, принимает радиопакеты от трекера и отдаёт их клиенту через COM-порт (USB) |

Клиент работает в двух режимах получения данных:

- **Локальный (COM-порт)** — приёмник SX1280/LoRa подключён к ПК напрямую; порты обнаруживаются автоматически в фоне, выбор — из меню. Сервер не требуется.
- **Сетевой (Track Server)** — подключение к RAJAGP Server по WebSocket (нативный WinHTTP, без сторонних сокет-библиотек, поэтому работает и на ARM64). Сервер присылает геометрию трека, состояния машин и авторитетные тайминги; клиент отслеживает качество канала (потери пакетов, джиттер).
- Дополнительно есть **встроенный симулятор** — генерирует машины на треке для тестирования без железа.

## Функционал

### Визуализация
- Карта трека на OpenGL 3.3: асфальт, бордюры, сетка 20 м, камера с зумом/поворотом/инерцией.
- Машины в реальном времени с интерполяцией движения, подписями имён и таймаутом пропавших трекеров.
- Тёмная тема UI (Dear ImGui), кастомные шрифты и иконки.

### Хронометраж и гонки
- Круги, лучший круг, дельта (к лучшему / предыдущему / конкретному кругу — настраивается).
- Секторы (S1/S2/S3) с цветовой индикацией.
- Режимы гонки: **Circuit Race**, **Time Attack**, **Rally**; фазы: Practice → Race → Finishing → Finished.
- Флаги гонки (green / yellow / red / finish) — локально или синхронизированно с сервера.
- Старт/финиш/автостоп гонки, экспорт результатов в `.txt` (**Ctrl+S**) и печать (**Ctrl+P**).

### PRO-режим (Pit Wall)
Отдельный экран из плавающих панелей (`src/ui/pro/`), с блокировкой раскладки и независимым зумом текста каждой панели (Ctrl+колесо):

- **Laptime** — крупный текущий круг и дельта;
- **Lap List** — список кругов с секторами;
- **Lap Info** — сводка по кругу;
- **Sectors** — детализация секторов;
- **Track Map** — миникарта с позицией машины;
- **G-Force** — диаграмма перегрузок;
- **Channels** — телеметрические каналы (скорость и др.);
- **Events** — лента событий гонки.

Масштабы каналов сохраняются в `pro_scales.ini`, раскладка — в `imgui.ini`.

### Создание треков
- **Запись по телеметрии**: машина с трекером проезжает круг — клиент пишет траекторию, замыкает петлю и сглаживает сплайном.
- **Двухконтурный алгоритм (dual-edge)**: проезд по левому краю, затем по правому — получается трек с реальными границами. Хранится в бинарном формате `.trk2` (заголовок + два полилайна, UTM-привязка через GeographicLib).
- Сохранение/загрузка треков (`saves/`), передача трека по сети чанками.

### Доступ и аккаунты
- Роли **user** / **admin** (токен вводится при подключении). Админ управляет гонкой и выдаёт доступы: меню **Networking → Accounts** — токены на 1 час / 1 день / бессрочно, копирование кликом.

## Технологии

C++17 · OpenGL 3.3 (glad) · GLFW · Dear ImGui · GLM · GeographicLib (UTM/геодезия) · serialib (COM) · WinHTTP WebSocket · nlohmann/json · stb_image

## Сборка

### Требования
- Windows 10/11 (x64 или ARM64)
- Visual Studio с тулсетом **v145** и компонентами C++
- [vcpkg](https://vcpkg.io) (`C:\vcpkg`) с пакетами `glfw3` и `geographiclib` под нужный триплет (`x64-windows` / `arm64-windows`)
- Репозиторий **RAJAGP Server** рядом с клиентом — клиент подключает общие заголовки протокола `rajagp_core` по относительному пути:

```
source/repos/
├── OpenGL-BONI-MVP-/     ← этот репозиторий (клиент, ветка ARM-dev)
└── RAJAGP Server/        ← сервер + rajagp_core (общий протокол)
```

Без соседнего каталога `RAJAGP Server` проект не соберётся.

### Шаги
1. Открыть `OpenGL.sln` в Visual Studio.
2. Выбрать конфигурацию `Release` (или `Debug`) и платформу `x64` / `ARM64`. Триплет vcpkg выбирается автоматически по платформе.
3. Build. Бинарь: `x64\Release\OpenGL.exe` (или `ARM64\Release\OpenGL.exe`).

> На ARM-машинах (Windows on ARM) собирайте и запускайте **ARM64**-конфигурацию — x64-сборка с arm64-зависимостями vcpkg падает с 0xc000007b.

### Инсталлятор
`installer.iss` — скрипт Inno Setup: пакует `x64\Release` (exe + все DLL, включая VC++ CRT), стили, Mesa3D-фоллбек (программный OpenGL для VM/RDP/старых GPU) и при необходимости ставит VC++ Redistributable. Готовый установщик: `Installer\BONI-Setup-x64.exe`.

## Использование

1. Запустить `OpenGL.exe`.
2. Источник данных:
   - **COM**: выбрать порт приёмника в меню (обнаруживается автоматически);
   - **Сервер**: **Networking → Connect to Server** (Shift+C) — адрес `IP:8080` и токен.
3. Загрузить или записать трек, запустить гонку из меню **Race** (админ).

### Горячие клавиши

| Клавиша | Действие |
|---|---|
| **F1** | Справка по всем горячим клавишам |
| **Shift+C** | Подключение к серверу |
| **Ctrl+S** | Сохранить результаты в `.txt` |
| **Ctrl+P** | Печать результатов |
| **Ctrl+колесо** | Зум текста панели (PRO-режим) |
| **Esc** | Выход |

## Структура проекта

```
OpenGL/src/
├── core/        главный цикл, инициализация GLFW/OpenGL
├── network/     COM-приёмник (ESP32_Code), WebSocket-клиент Track Server, симулятор
├── racing/      RaceManager, режимы/фазы гонки, старт-стоп, дельты
├── rendering/   отрисовка трека и машин, интерполяция, подписи
├── track/       запись трека по телеметрии, построение сплайнов, .trk2
├── ui/          главный UI, аккаунты, PRO-панели (ui/pro/)
└── vehicle/     состояние машин
```
